## How to use this wiki

- **Read** pages just like any other doc — they render straight from Markdown.
- **Edit** any page by clicking **Edit page** in the top-right panel. You&rsquo;ll be prompted for the shared editor password the first time.
- **Add a new page** with the **New page** button. Pick a section, give it a title, and start writing.
- **Roll back a mistake** by going to the GitHub repo &rsquo;s commit history.

## Sections

- **Overview** — high-level org info, links, roster
- **Operations** — repeatable processes (onboarding, events, meetings)
- **Communities** — the seven topical sub-communities within MDplus
- **Admin** — credentials, legal docs, finance

## Editing tips

- Use `## Heading` for top-level sections — they show up in the right-hand TOC.
- Use `<Callout>`, `<LinkCard>`, and `<PersonRow>` to add structure beyond plain Markdown.
- Every save creates a GitHub commit and triggers a Vercel redeploy (~30 seconds).
